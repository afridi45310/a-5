Question-3: What is the difference between innerHTML, innerText, and textContent ?

1.innerHtml দিয়ে শৈলী বা স্ক্রিপ্ট এলিমেন্ট ছাড়া CSS লুকানো টেক্সট স্পেসিং এবং ট্যাগ ছাড়াই এলিমেন্টের টেক্সট কন্টেন্ট ব্যবহার করা যায়। 2.innerText এই বৈশিষ্ট্যটি রেন্ডার করা টেক্সট কন্টেন্টের উপর ফোকাস করে। যখন কোনও এলিমেন্টের কন্টেন্ট পড়ার জন্য innerText ব্যবহার করেন, তখন এটি স্ক্রিনে প্রদর্শিত টেক্সটটি যেমনটি দেখা যায় তেমনই ফেরত দেয়। এটি HTML ট্যাগগুলিকে উপেক্ষা করে। এবং এতে CSS স্টাইলের সাথে লুকানো টেক্সটও অন্তর্ভুক্ত থাকে না। 3.textContent প্রপার্টিটি সমস্ত HTML ট্যাগ উপেক্ষা করে এবং শুধুমাত্র টেক্সট রিটার্ন করে। যদিও innerText স্ক্রিনে রেন্ডার করা টেক্সটটি পড়ে, textContent মার্কআপে থাকা টেক্সটটি পড়ে। এটি স্ক্রিনে রেন্ডার করা হোক বা না হোক, সমস্ত টেক্সট রিটার্ন করে।

Question-3: What is event delegation in the DOM?

ইভেন্ট ডেলিগেশন বলতে বোঝায় ইভেন্ট প্রোপাগেশন (বাবলিং) ব্যবহার করে ইভেন্টের উৎপত্তিস্থলের উপাদানের চেয়ে DOM-এ উচ্চ স্তরে ইভেন্ট পরিচালনা করার প্রক্রিয়া। এটি বর্তমানে বা ভবিষ্যতে বিদ্যমান উপাদানগুলির জন্য একটি একক ইভেন্ট লিসেনার সংযুক্ত করার অনুমতি দেয়। ইভেন্ট হ্যান্ডলিং ফাংশনের ভিতরে।

Question-4: What is event bubbling in the DOM?

ইভেন্ট বাবলিং হল এক ধরণের DOM ইভেন্ট প্রোপাগেশন যেখানে ইভেন্টটি প্রথমে সবচেয়ে ভেতরের টার্গেট এলিমেন্টে ট্রিগার হয় এবং তারপর একই নেস্টিং হায়ারার্কিতে টার্গেট এলিমেন্টের পূর্বপুরুষদের (পিতামাতাদের) উপর ধারাবাহিকভাবে ট্রিগার হয় যতক্ষণ না এটি বাইরেরতম DOM এলিমেন্ট বা ডকুমেন্ট অবজেক্টে পৌঁছায় ।

Question-5: How do you create, add, and remove elements using JavaScript?

একটি সূচক হল অ্যারেতে একটি উপাদানের অবস্থান। অ্যারের প্রথম উপাদানটির সূচক 0, দ্বিতীয় উপাদানটির সূচক 1, ইত্যাদি। একটি নির্দিষ্ট সূচকে উপাদান যোগ করতে, স্প্লাইস পদ্ধতি ব্যবহার করতে পারেন। স্প্লাইস পদ্ধতি একটি অ্যারেতে একটি নির্দিষ্ট সূচক থেকে উপাদান যোগ করতে এবং অপসারণ করতে দেয়।
